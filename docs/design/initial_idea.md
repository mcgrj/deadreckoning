# DARK EXPEDITION – GAME DESIGN DOCUMENT

## Working Title
(See name ideas at bottom)

---

# 🎯 HIGH CONCEPT

A **simulation-driven narrative roguelike** where the player commands a doomed maritime expedition in a harsh, fictionalised age-of-sail world inspired by real historical disasters like the Wager.

Each run generates a **self-contained tragic story** driven by:
- crew psychology
- scarcity
- leadership decisions (card system)
- emergent simulation

---

# 🧭 DESIGN PILLARS

## 1. Human Conflict First
The core gameplay is managing **people under pressure**, not combat.

## 2. Emergent Storytelling
Stories arise from systems, not scripts.

## 3. Meaningful Tradeoffs
Every decision solves one problem and creates another.

## 4. Replayable Tragedy
Each run is a complete arc: control → fracture → collapse.

---

# 🌍 SETTING

## Approach
- Fictional world
- Strongly inspired by real expeditions (e.g. Wager)
- Not tied to real events

## Tone
- Bleak
- Grounded
- Gritty realism
- Moral ambiguity

## Core Theme
> Authority collapses under pressure.

---

# 🔄 CORE LOOP

1. Select ship upgrades
2. Select crew
3. Begin expedition
4. Navigate map (like Slay the Spire)
5. Simulation ticks between nodes (e.g. squares traversed between nodes = simulation tick)
6. Events trigger (micro or major)
7. Player uses cards + crew
8. Systems evolve
9. Expedition ends (failure or survival)
10. Ship log generated

---

# 🗺️ MAP STRUCTURE

## Routes
- Coastal (safe, slow decay)
- Open ocean (fast, dangerous)
- Unknown waters (high variance)

## Nodes
- Distance squares (simulation ticks)
- Event nodes (rooms)

---

# ⚙️ SIMULATION SYSTEM

## Core Stats
- Food
- Morale
- Loyalty
- Health

## Social Systems
- Relationships
- Factions (emergent)
- Player reputation

## Tick System
Each square:
- updates resources
- updates relationships
- checks thresholds

---

# 👥 CREW SYSTEM

## Attributes
- Visible traits
- Hidden traits
- Needs
- Relationships

## Example Traits
- Loyal
- Greedy
- Superstitious
- Ambitious

## Design Goal
Player selects crew = selects problems

---

# ⚓ SHIP UPGRADES

## Philosophy
No pure upgrades

## Examples
- Reinforced Hull (safe vs slow)
- Strict Discipline (order vs rebellion)
- Expanded Storage (more food vs theft risk)

---

# 🃏 CARD SYSTEM

## Concept
Cards represent leadership actions.

## Categories
- Authority
- Social
- Risk
- Desperate

## Design Rules
- Context-sensitive
- Modify probabilities
- Not flat bonuses

---

# 🚪 EVENT SYSTEM

## Types
- Crisis
- Social
- Opportunity
- Breakdown

## Structure
1. Setup
2. Escalation
3. Player action
4. Resolution

---

# 🧪 SIMULATION + EVENTS

Events are triggered by conditions, not random.

Example:
Low food + low morale → theft event

---

# 📖 SHIP LOG

## During Run
Records key events

## End of Run
Generates narrative summary

## Meta Layer
Future runs reference past events

---

# 🔁 META PROGRESSION

## Unlocks
- Crew archetypes
- Traits
- Cards
- Events
- Starting conditions

## Rule
Adds variety, not power

---

# 🎯 RUN STRUCTURE

## Length
45–60 minutes

## Acts
1. Control
2. Fracture
3. Collapse

---

# ⚖️ DESIGN CONSTRAINTS

- Keep systems readable
- Avoid over-complexity
- Limit event frequency
- Maintain pacing

---

# 🧭 GAME IDENTITY

A **gritty, emergent narrative roguelike** about leadership, survival, and collapse under pressure.
